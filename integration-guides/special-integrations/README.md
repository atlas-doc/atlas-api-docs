# Special Integrations

{% include "../../.gitbook/includes/eva-help-hint.md" %}

Use this section only when your integration requires special airline workflows or resident fare logic.

Start here when you need to:

* implement airline-specific booking rules
* support resident fare eligibility or discount logic
* handle a business case that does not fit the standard booking flow

### FAQ

#### When should I use Special Integrations?

Use this section only when the standard Atlas booking flow is not enough and the business case needs airline-specific or discounted-fare logic.

#### Which special integration page should I open first?

Use **FR Integration** for Ryanair-specific confirmation and payment handling.

Use **Resident Price Integration** for Spain resident fare eligibility and discount flows.

### Pages in this section

* [FR Integration](fr-integration.md)
* [Resident Price Integration](resident-price-integration.md)

### Use this when you need

* Ryanair confirmation and payment flows
* Resident fare eligibility and discount logic
* Airline-specific compliance requirements

### What comes next?

After you confirm the business case fits a special flow, open the matching guide and follow that workflow instead of the default booking path.
